Checklist Series 01 — IC / AP-Memory Placement
CKL-01-001 ICs must not overlap with interposers or connectors on the opposite side Placement ▶

Inspection Process

  1. Search for all ICs on both Top and Bottom layers.
  2. For each IC, check if any interposer or connector component overlaps on the opposite side.
  3. Report overlapping pairs.

Output Columns

ColumnDescription
compName of the IC component
cmp_layerLayer of the IC (Top or Bottom)
overlapping_cmpName of the overlapping interposer or connector on the opposite side
statusPASS or FAIL

Pass/Fail Criteria

FAIL if any interposer or connector overlaps the IC on the opposite side.
CKL-01-005 Inductors (≥2012) overlapping opposite side of AP/Memory: corner placement and orientation review Placement ▶

Reference Files

references/ap_memory.csv references/inductors_2s_list.csv

Inspection Process

  1. Search for AP or Memory components on Top and Bottom layers (part names from ap_memory.csv).
  2. Search for all inductors overlapping the opposite side of each AP/Memory.
  3. Filter inductors to include only those with a size of 2012 or larger.
  4. Inspect whether filtered inductors are placed at the corners and check their horizontal/vertical orientation.

Output Columns

ColumnDescription
compName of the AP or Memory component
cmp_layerLayer of the AP/Memory (Top or Bottom)
overlapping_indName of the overlapping inductor (≥2012)
part_namePart name of the inductor
edgeTRUE if inductor is at the corner of the AP/Memory, FALSE otherwise
hori/vertiOrientation relative to the AP/Memory outline (Horizontal or Vertical)
statusPASS or FAIL

Pass/Fail Criteria

PASS if edge is FALSE and orientation is Horizontal. Otherwise FAIL.
Checklist Series 02 — Capacitor / Inductor vs Connector
CKL-02-001 10 managed capacitor types must be at least 1.5mm from connectors on the opposite side Spacing ▶

Reference Files

references/capacitors_10_list.csv

Inspection Process

  1. Search for 10 managed capacitor types on Top and Bottom layers.
  2. Match each identified capacitor with any connector on the opposite side.
  3. If an opposite-side connector is found, measure the edge-to-edge distance.
  4. Distance must be at least 1.5mm.

Output Columns

ColumnDescription
compName of the managed capacitor
cmp_layerLayer of the capacitor
part_namePart name of the capacitor
overlapping_conName of the connector on the opposite side
distanceEdge-to-edge distance (mm)
statusPASS or FAIL

Pass/Fail Criteria

PASS if distance ≥ 1.5mm. Otherwise FAIL.

CKL-02-012 Inductors ≥2012 overlapping both AP/Memory and Shield Can on opposite sides must be avoided Placement ▶

Reference Files

references/ap_memory.csv references/inductors_2s_list.csv

Inspection Process

  1. Identify AP/Memory components on each layer using part names from ap_memory.csv.
  2. Find inductors on the opposite side overlapping each AP/Memory component.
  3. Filter inductors to size ≥ 2012 using inductors_2s_list.csv.
  4. For each qualifying inductor, check whether a Shield Can on the same side as the AP/Memory also overlaps the inductor.
  5. Report any inductor that simultaneously overlaps both an AP/Memory and a Shield Can.

Output Columns

ColumnDescription
compName of the inductor
cmp_layerLayer where the inductor is placed
part_namePart name of the inductor
overlapping_apName of the AP/Memory component on the opposite side
statusPASS or FAIL

Pass/Fail Criteria

FAIL if an inductor (≥2012) overlaps both an AP/Memory component and a Shield Can simultaneously.
Checklist Series 03 — OSC / MIC / Component Placement & Clearance
CKL-03-001 Connectors must not overlap the opposite side of MCP IC components Clearance ▶

Reference Files

references/ap_memory.csv

Inspection Process

  1. Identify MCP (Multi-Chip Package) ICs on each layer by looking up the category column in ap_memory.csv.
  2. Search for connectors on the opposite side of each MCP IC.
  3. Report any connector that overlaps the footprint of an MCP IC.

Output Columns

ColumnDescription
compName of the MCP IC component
cmp_layerLayer of the MCP IC (Top or Bottom)
overlapping_cmpName of the overlapping connector on the opposite side
part_namePart name of the connector
statusPASS or FAIL

Pass/Fail Criteria

FAIL if any connector overlaps an MCP IC on the opposite side.
CKL-03-012 OSC components must be at least 1mm from PCB edge and BOTHHOLE components Clearance ▶

Inspection Process

  1. Collect all BOTHHOLE components from both Top and Bottom layers.
  2. Build the board outline polygon from the PCB profile data.
  3. For each Oscillator (OSC) component, measure the distance to the PCB outline and the distance to the nearest BOTHHOLE.
  4. Both distances must be at least 1mm.

Output Columns

ColumnDescription
compName of the OSC component
cmp_layerLayer where the OSC is placed (Top or Bottom)
to_pcbEdge-to-edge distance to the PCB outline (mm), or N/A if not measurable
to_BTHEdge-to-edge distance to the nearest BOTHHOLE component (mm), or N/A if none present
statusPASS or FAIL

Pass/Fail Criteria

PASS if both to_pcb ≥ 1mm and to_BTH ≥ 1mm. Otherwise FAIL.
CKL-03-013 Each MIC pad must have at least one VIA Placement ▶

Inspection Process

  1. Search for MIC components (comp_name starts with MIC) on Top and Bottom layers.
  2. For each MIC component, iterate over all pads defined in the package.
  3. Transform each pad centre from package-local coordinates to board coordinates.
  4. Count the number of VIAs whose board position falls within 0.05mm of the pad centre.
  5. Report one row per pad with the VIA count and pass/fail result.

Output Columns

ColumnDescription
compName of the MIC component
cmp_layerLayer where the MIC component is placed (Top or Bottom)
padName / number of the specific pad
viaNumber of VIAs located at that pad position
statusPASS or FAIL

Pass/Fail Criteria

FAIL if the VIA count for a pad is 0. PASS if 1 or more VIAs are present.
CKL-03-015 Components must have at least 0.65mm clearance from the PCB outline Clearance ▶

Inspection Process

  1. Generate a reference boundary line 0.65mm inward from the PCB outline.
  2. For the Top layer, identify any component pins or pads within the clearance zone (between the PCB outline and the reference line).
  3. Calculate the direct linear distance between the identified components and the PCB outline.
  4. Perform the same process for the Bottom layer.

Output Columns

ColumnDescription
compName of the component
part_namePart name of the component
cmp_layerLayer where the component is placed
distanceMeasured distance to the PCB outline (mm)
statusPASS or FAIL

Pass/Fail Criteria

PASS if distance ≥ 0.65mm. Otherwise FAIL.
CKL-03-016 Interposers, Shield Cans, SIM Sockets, and Connectors must not overlap the opposite side of OSC components Placement ▶

Inspection Process

  1. Search for Oscillator (OSC) components on the Top and Bottom layers.
  2. For each OSC, collect all Interposers (INP), Shield Cans (SC), SIM Sockets (SIM), and Connector components on the opposite side.
  3. Check if any of those components overlap the OSC footprint.
  4. Report one row per overlapping component; OSCs with no overlap are recorded as PASS.

Output Columns

ColumnDescription
compName of the OSC component
cmp_layerLayer where the OSC is placed (Top or Bottom)
overlapping_cmpName of the overlapping component on the opposite side (INP, SC, SIM, or Connector), or “–” if none
part_namePart name of the overlapping component, or “–” if none
statusPASS or FAIL

Pass/Fail Criteria

FAIL if any INP, SC, SIM, or Connector overlaps the opposite side of an OSC component.

Architecture Notes

Adding New Checklist Rules

  1. Create a new file in src/checklist/rules/ (e.g. ckl_04_001.py).
  2. Define a class that extends ChecklistRule with rule_id, description, category, and an evaluate() method.
  3. Decorate the class with @register_rule.
  4. Return a RuleResult with details={"columns": [...], "rows": [...]} for automatic tabular Excel output.
  5. Add the import to main.py inside cmd_check().
