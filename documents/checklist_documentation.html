체크리스트 시리즈 01 — IC / PMIC / 안테나 / 센서 배치
CKL-01-001 IC 및 Filter는 반대면의 Interposer / Connector / SIM Socket / Shield Can과 패드-패드 겹침이 없어야 함 Placement ▶

검사 과정

  1. Top 및 Bottom 레이어에서 모든 IC 및 Filter를 탐색한다.
  2. 각 IC에 대해 반대면의 Interposer(최외곽 패드만), Connector / SIM Socket / Shield Can(모든 패드) 과의 패드 겹침을 검사한다.
  3. 각 Filter에 대해 반대면의 Interposer(최외곽 패드만), Connector / SIM Socket(모든 패드) 과의 패드 겹침을 검사한다.
  4. 외곽선만 겹치는 경우는 PASS로 기록하고, 패드-패드 접촉이 발생하면 FAIL로 플래그한다.

출력 컬럼

컬럼설명
compIC 또는 Filter 부품 이름
cmp_layer부품이 배치된 레이어 (Top 또는 Bottom)
overlapping_cmp반대면에서 겹치는 부품 이름
opp_type반대면 부품의 유형 (Interposer / Connector / SIM_Socket / Shield_Can)
statusPASS 또는 FAIL

Pass/Fail 기준

FAIL: 반대면의 대상 부품 패드와 IC/Filter 패드가 겹치면 FAIL.
CKL-01-002 PMIC 부품의 최외곽 패드에는 VIA 설계가 적용되어야 함 Placement ▶

참조 파일

references/pmic_list.csv

검사 과정

  1. pmic_list.csv 의 part_name 및 속성 기반 분류를 통해 PMIC 부품을 식별한다.
  2. 각 PMIC 패키지의 핀 배열 중 외곽 경계에 위치한 최외곽 패드 인덱스를 계산한다.
  3. 각 최외곽 패드 위치에서 VIA 수를 카운트한다 (toeprint 및 FID 기반 실제 동박 패드 geometry 사용).
  4. 패드가 NC(Not Connected) 인 경우는 FAIL 판정에서 제외한다.

출력 컬럼

컬럼설명

pad_name패드(핀) 이름
via해당 패드 위치에 VIA가 존재하는지 여부 (TRUE / FALSE)
ncNC 패드 여부 (TRUE / FALSE)
statusPASS / FAIL / NC

Pass/Fail 기준

FAIL: 최외곽 패드에 VIA가 없고 NC 패드가 아닌 경우 FAIL. NC 패드는 판정에서 제외된다.
